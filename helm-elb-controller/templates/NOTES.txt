This cluster is now ready for app charts.

To expose an app:
1) Create an Ingress with:
   - spec.ingressClassName: {{ .Values.ingressClass.name }}
   - Proper annotations for ALB (example below)
2) (Optional) Add external-dns annotations if you prefer annotations over hostname rules.

Example Ingress:

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: myapp
  annotations:
    alb.ingress.kubernetes.io/listen-ports: '[{"HTTP":80},{"HTTPS":443}]'
    alb.ingress.kubernetes.io/scheme: {{ .Values.ingressClass.params.scheme }}
    alb.ingress.kubernetes.io/target-type: {{ .Values.ingressClass.params.targetType }}
    alb.ingress.kubernetes.io/ssl-redirect: "443"
    # external-dns (either here or rely on rules.hosts)
    external-dns.alpha.kubernetes.io/hostname: myapp.example.com
spec:
  ingressClassName: {{ .Values.ingressClass.name }}
  rules:
    - host: myapp.example.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: myapp-svc
                port:
                  number: 80
